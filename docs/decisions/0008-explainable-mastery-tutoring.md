# ADR-0008：首版辅导采用教师要点规则与累计掌握度

- 状态：已采纳
- 日期：2026-07-14

## 决策

个性化辅导不使用不可解释的推荐或生成评分模型。教师维护版本化知识点目录，每个知识点包含来源检索词、题目模板和若干评分要点；每个要点列出允许的关键词或同义表达。系统按“显式指定知识点 → 查询别名匹配 → 最低掌握度”选择题目，检索课程资料并保存引用，然后创建练习实例。

提交答案后，每个评分要点等权计分，返回已覆盖和待补充项。知识点掌握度采用该学生历次得分的累计平均值：低于 50 为 `needs_review`，50–79.99 为 `developing`，80 以上为 `proficient`。学生答题前看不到评分要点；班级接口只返回知识点级人数、均分和状态计数。

## 原因

- 评分结果能逐项解释、固定复现，适合当前课程资料规模。
- 练习实例保存资料引用，避免脱离教材随意出题。
- 不调用昂贵模型，固定评测和本地开发都可稳定运行。
- 累计平均值规则简单透明，后续可以在教师评测支持下替换。

## 代价

- 关键词规则无法识别所有正确改写，也不能评价复杂操作题的步骤质量。
- 等权评分没有体现关键安全步骤更高的重要性。
- 当前班级范围由请求角色表达；接入学校身份系统前不能用于真实成绩管理。

## 验收

固定辅导任务集在隔离数据库中端到端执行练习生成、来源追溯、批改和掌握度更新；实测报告 `tutoring_eval_20260714T130857Z.json` 的 4 条用例全部通过，平均绝对分数误差为 0。
