# ADR-0007：bad case 经教师断言后才能晋升回归集

- 状态：已采纳
- 日期：2026-07-14

## 决策

负向反馈和运行异常自动创建 bad case，但不能直接把原回答当成正确金标。教师或维护者必须先填写可执行断言，再晋升为回归用例：

- 预期最终状态、任务类型和风险等级。
- 答案必须包含与不得包含的短语。
- 是否必须返回引用。
- 审核说明和 bad case 处理状态。

导出包包含脱敏的多轮输入、原结果、反馈、配置、Trace 和断言，并以稳定 `package_hash` 标识。重放复制当前知识快照和结构化报警表到隔离数据库，不修改正式运行记录。

## 原因

- 用户给差评只说明结果不满意，不等于原状态或原答案是期望行为。
- 自动回归必须有机器可判定的验收条件，不能依赖比较整段生成文本。
- 多轮上下文、检索配置和知识快照都会影响结果，必须随用例记录。
- Trace 和反馈可能含个人信息，进入 CourseOps 前必须递归脱敏。

## 代价

- 每条回归用例需要人工审核，不能完全自动晋升。
- 当前同步重放会复制知识快照；大规模回归需要独立 worker 和资源配额。
- 文本断言不能完全代表答案忠实度，后续仍需教师评分量表或受控语义评估。

## 验收

真实服务闭环创建了 1 条负反馈 bad case，审核后导出稳定哈希包，隔离重放通过 7 项断言并返回 5 条引用；晋升后的回归报告 `regression_20260714T124351Z_14624ebb.json` 为 1/1 通过。
