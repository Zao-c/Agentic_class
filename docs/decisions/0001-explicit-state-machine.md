# ADR-0001：使用显式有限状态机

- 状态：已采纳
- 日期：2026-07-14

## 决策

第一版不使用提示词驱动的自由 Agent 循环，也不把业务正确性绑定到 LangGraph。节点和转移由 Python 显式实现，状态使用 Pydantic 序列化并持久化到 SQLite。

## 原因

- 故障诊断必须保证先收集关键槽位，再检索和判断安全。
- 最大步数、拒答和人工转交条件需要可测试。
- 运行不依赖外部模型，便于建立稳定回归基线。
- 后续可以把相同节点映射到 LangGraph，而不改变领域状态和 Trace Schema。

## 代价

规则意图识别和模板生成能力有限；接入模型后仍须保留结构化校验和确定性转移。
