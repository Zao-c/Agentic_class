# 已知限制与技术债

更新时间：2026-07-15

## P0：进入真实教学试点前必须完成

- 当前角色由 `X-Role` / `X-User-ID` 表达，只适合本地演示；生产必须接入学校身份认证并由服务端签发身份。
- 现有 `data/active` 未完成逐份版权、有效期、设备型号和班级访问范围复核。
- 当前结构化报警码库只有从课程资料核验出的 ABB `10036` 和 `38213` 两条记录，且原始资料未标明具体型号。进入真实试点前必须由教师以指定设备官方手册补齐型号、控制器版本、页码和审核状态。
- 进程内 BackgroundTasks 不具备崩溃恢复能力；生产应使用持久任务队列或启动恢复器。

## P1：任务书阶段 1–3

- 新增的 180 条检索任务和 10 份微型教材全部是公开原创合成数据。它们可以验证规模化评测、split、防泄漏、检索指标和报告发布，但模拟教师/学生不构成真实审核或学员证据；当前教师确认 Gold 仍为 0。

- 当前开发机未安装 Docker CLI，因此本轮无法执行 `docker compose config/build/up`；已提供 CI 构建/健康任务与 `scripts/accept_docker.py` 完整验收入口，仍需在真实 Docker 主机补做三任务、状态卷和重启恢复验收。
- 已接入版本锁定的 BGE Embedding 与真实 Cross-Encoder，并在 12 条课程样本上完成同口径消融；但样本量不足以代表生产准确率，仍需用教师确认的真实 bad case 扩充评测集。可解释特征精排与 Cross-Encoder 在报告和 Trace 中保持不同名称。
- 神经模型缓存位于项目目录之外，当前 Docker 配置没有安装神经依赖或挂载模型缓存；容器默认只能运行 `portable` 档，神经档部署需要单独镜像和只读模型卷。
- `neural-quality` 在当前 CPU 环境的检索 P50 约 3.27 秒，不适合作为默认在线链路；尚未完成并发压测、内存上限和 GPU 部署评估。
- Windows CPU 上的 PyTorch 首次工作线程推理可能耗时数十秒；神经档已把真实推理预热纳入就绪门槛，因此首次启动较慢但不会把初始化延迟转嫁给首个学生请求。
- 工具已接入统一超时、重试与熔断器；模型调用有独立超时和有界重试，但仍需在真实供应商限流、断网和返回畸形 JSON 的条件下压测。
- 未接入受控 WebSearch。允许域名、资料时效、引用和失败策略确定前保持关闭。
- 二进制文档解析保留文字顺序，但表格、页码和 PPT 视觉结构尚未完成专项 QA。
- Agentic Trace 已记录模型 Token、估算成本、耗时、重试和 fallback，但还没有 OpenTelemetry/LangSmith；成本只有在配置供应商单价后才有意义。
- 已完成 DeepSeek V4 Flash 单条真实 HTTP 烟测，建立 12 条冻结工程比较集与 8 条聊天红队集，并执行 portable 单次基线；自由 Agent、受控 Agent 多次运行和正式 Gold 三方案成本/P95 对比仍未执行。Agentic 调用会把经裁剪的请求与证据发送给所配置供应商，真实教学前必须完成数据出境、隐私和供应商保留策略评审。
- portable 红队首次运行确认了虚假型号、多轮撤回、高风险意图和冲突证据四类问题；本轮已用通用规则修复并为 portable/Agentic 增加回归，同一 8 条集由 0.50 提升为 1.00。6 类系统故障注入已用确定性离线 fake 执行并保留公开报告，但尚未执行真实供应商超时/429、学校身份认证、OS 沙箱或机器人控制链路验收。聊天样本规模仍不足，必须继续增加变体与跨语言表达。
- 旧评测集的预期来源路径来自历史项目；兼容逻辑只比较文件名，需迁移为当前 `document_id`。
- 故障任务集目前只有 7 条工程验收用例，指标用于验证路径而非宣称诊断准确率；同码异义已由自动化测试覆盖，但还缺少真实厂商跨型号样本。
- bad case 重放和回归已支持本机同步执行，但尚未接入 CI 定时任务、持久队列或并发资源配额；大量用例或神经档回放应迁移到独立 worker。
- 当前回归断言以状态、风险、文本包含/排除和引用存在性为主，尚未加入教师答案评分量表或语义忠实度模型。

## P2：个性化辅导

- 已实现版本化知识点、资料可追溯练习、确定性批改、逐项解释和知识点掌握度；但关键词规则无法可靠评价复杂自由表述、操作质量或步骤间因果关系，不能作为正式考试评分器。
- 当前要点等权、掌握度使用累计平均值，尚未加入安全关键项权重、难度校准、遗忘曲线或教师人工复核工作台。
- 学习数据可查看并删除练习、作答、掌握度和学习记录，但该接口不会删除 Run、事件、反馈或 bad case；尚未实现全量主体访问/删除、单项删除、数据导出、自动保留期限和学校班级花名册授权策略。
